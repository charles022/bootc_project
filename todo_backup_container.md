# PROPOSAL: Decouple `backup-container` from Pod Orchestration

## Status: Proposed
**Problem:** The `backup-container` is currently implemented as a sidecar within the `devpod` manifest. This couples the backup infrastructure to the development workload, violating the design goal of an "invisible," host-managed backup service. It also forces the backup container to share a network/IPC namespace with the development environment, which is unnecessary and reduces isolation.

### 2. Host-Managed Service (The "Invisible" Layer)
*   **NEW: `01_build_image/build_assets/backup.container`**:
    *   **Content**: A new Quadlet file defining a standalone container. No `[Install]` section is needed â€” this unit is activated only by `backup.timer`, not at boot directly.
    ```ini
    [Unit]
    Description=Host-managed data backup service
    
    [Container]
    Image=quay.io/m0ranmcharles/fedora_init:backup-container
    # Mount host volumes for backup (e.g., tenant data root)
    Volume=/var/lib/openclaw-platform:/var/lib/openclaw-platform:ro
    Exec=/usr/local/bin/backup_stub.sh
    ```
*   **NEW: `01_build_image/build_assets/backup.timer`**:
    *   **Content**: A systemd timer to handle the schedule. Activates `backup.service` by naming convention (Quadlet generates `backup.service` from `backup.container`).
    ```ini
    [Unit]
    Description=Daily backup trigger
    
    [Timer]
    OnCalendar=daily
    Persistent=true
    
    [Install]
    WantedBy=timers.target
    ```

## Rationale
*   **Isolation**: Dev environment crashes no longer stop backups; backup failures no longer crash the dev environment.
*   **Security**: The backup service can have read-only access to `/var/lib/openclaw-platform` which the `dev-container` does not need and should not see.
*   **Efficiency**: The host can schedule backups during low-usage windows regardless of whether the dev pod is active.

## Verification Plan
1.  **Build**: Verify `build_image.sh` produces all images without errors.
2.  **VM Test**: Boot the image and run `podman ps --pod`. Only `dev-container` should be present in the pod.
3.  **Service Test**: Run `systemctl start backup.service` and verify logs via `journalctl -u backup.service`.
4.  **Timer Test**: Run `systemctl list-timers` to confirm `backup.timer` is correctly registered.
